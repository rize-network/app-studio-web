# App Studio Web Component Library

A comprehensive, accessible, and customizable React component library built with TypeScript and App Studio. This library provides 60+ production-ready UI components designed for building modern web applications with full type safety, accessibility support, and seamless form integration.

## Overview

App Studio Web is a complete React component library featuring:
- **TypeScript-first**: Built with TypeScript 4.9+ for complete type safety and IntelliSense support
- **40+ Core UI Components**: Comprehensive set of layout, form, navigation, feedback, and data display components
- **ADK Integration**: Specialized components for Agent Development Kit (ADK) integration
- **Formik Integration**: Seamless form handling with dedicated Formik wrapper components
- **Accessible**: WCAG-compliant components with proper ARIA attributes and keyboard navigation
- **Themeable**: Flexible theming system with light/dark mode support and customizable design tokens
- **Production Ready**: Battle-tested components used in real-world applications
- **Open Source**: MIT licensed and ready for AI-assisted development

## Installation

### NPM
```bash
npm install @app-studio/web
# or

```

### Manual Installation
```bash
# Clone the repository
git clone https://github.com/your-org/app-studio-web.git
cd app-studio-web

npm install

# Build the library
npm run build
```

## Theming and Customization

### Dark Mode
App Studio Web includes built-in support for light and dark themes:

```tsx
import { ThemeProvider } from 'app-studio';

function App() {
  return (
    <ThemeProvider theme={{ mode: 'dark' }}>
      {/* Your app */}
    </ThemeProvider>
  );
}
```

Individual components can override the theme mode:
```tsx
<Button themeMode="dark">Dark Button</Button>
```

### Theme System

The library uses a comprehensive theme system with:

#### Color Palette
- **Semantic Colors**: `theme.primary`, `theme.secondary`, `theme.success`, `theme.warning`, `theme.error`, `theme.info`, `theme.disabled`
- **Neutral Colors**: `color.white`, `color.black`, `color.gray.50` through `color.gray.900`
- **Color Scales**: `color.blue`, `color.green`, `color.red`, `color.yellow`, `color.purple` (50-900)

#### Typography
- **Font Family**: Inter/Geist
- **Font Sizes**: 12px to 48px
- **Font Weights**: 400 (regular) to 700 (bold)
- **Line Heights**: 1.2 to 1.8

#### Spacing
Uses a 4px grid system:
- 4px (extra small)
- 8px (small)
- 16px (medium)
- 24px (large)
- 32px (extra large)

#### Shapes
- `sharp`: No border radius (0px)
- `rounded`: Small border radius (4px)
- `pillShaped`: Pill shape (9999px)

#### Shadows
- `shadow.sm`: Small shadow
- `shadow.md`: Medium shadow
- `shadow.lg`: Large shadow
- `shadow.xl`: Extra large shadow

## Forms and Validation

### Formik Integration

The library provides dedicated Formik wrapper components for seamless form handling:

```tsx
import { Formik, Form } from 'formik';
import { FormikTextField, FormikCheckbox } from '@app-studio/web';
import * as Yup from 'yup';

const schema = Yup.object({
  email: Yup.string().email().required(),
  terms: Yup.boolean().oneOf([true]).required(),
});

function MyForm() {
  return (
    <Formik
      initialValues={{ email: '', terms: false }}
      validationSchema={schema}
      onSubmit={(values) => console.log(values)}
    >
      <Form>
        <FormikTextField name="email" label="Email" type="email" />
        <FormikCheckbox name="terms" label="I accept the terms" />
      </Form>
    </Formik>
  );
}
```

## API Integration

The library includes comprehensive API integration support with OpenAPI-generated services:

### Generated API Hooks
```tsx
import { UserService } from 'src/services/api';

function UserProfile({ userId }) {
  const getUserRequest = UserService.useUserControllerReadService({
    onSuccess: (data) => {
      console.log('User loaded:', data);
    },
    onError: (error) => {
      console.error('Error:', error);
    },
  });

  useEffect(() => {
    getUserRequest.run(userId);
  }, [userId]);

  return (
    <div>
      {getUserRequest.loading && <Loader />}
      {getUserRequest.data && <div>{getUserRequest.data.name}</div>}
    </div>
  );
}
```

# Component Usage Guide for AI Agents

This section provides comprehensive guidance for AI agents on how to effectively use App Studio Web components.

## Core Principles

### 1. Import Pattern
Always import components from `@app-studio/web`:
```tsx
import { Button, Title, Card } from '@app-studio/web';
import { Vertical, Horizontal, Center } from 'app-studio';
```

### 2. Component Composition
Components are designed to be composed together:
```tsx
<Card variant="elevated" size="lg">
  <Card.Header>
    <Title size="lg" highlightText="Featured">Featured Product</Title>
  </Card.Header>
  <Card.Content>
    <Vertical gap={16}>
      <Text>Product description goes here</Text>
      <Button variant="filled">Add to Cart</Button>
    </Vertical>
  </Card.Content>
</Card>
```

### 3. Prop Naming Conventions
- **Boolean props**: Prefix with `is` (e.g., `isDisabled`, `isLoading`, `isFullWidth`)
- **Event handlers**: Prefix with `on` (e.g., `onClick`, `onChange`, `onSubmit`)
- **Style props**: Use semantic names (e.g., `variant`, `size`, `shape`, `shadow`)
- **Content props**: Direct names (e.g., `label`, `placeholder`, `header`, `footer`)

## Common Component Patterns

### Title Component
The Title component supports advanced text animations and highlighting:

**Basic Usage:**
```tsx
<Title size="xl">Simple Title</Title>
```

**With Highlighting:**
```tsx
<Title 
  size="lg"
  highlightText="Amazing"
  highlightStyle="gradient"
  highlightColor="theme.primary"
  highlightSecondaryColor="theme.secondary"
>
  This is Amazing
</Title>
```

**With Typewriter Effect:**
```tsx
<Title
  highlightText="Typewriter"
  highlightTypewriter={true}
  highlightTypewriterDuration={2000}
  highlightStyle="glow"
  highlightColor="theme.primary"
>
  Watch the Typewriter effect
</Title>
```

**With Slide Animation:**
```tsx
<Title
  highlightText={["Slide", "Effect", "Multiple"]}
  highlightSlide={true}
  highlightSlideSequential={true}
  highlightSlideStagger={200}
  highlightStyle="underline"
  highlightColor="theme.primary"
>
  Sequential Slide Effect Multiple
</Title>
```

**With Alternating Text:**
```tsx
<Title
  highlightText="Amazing"
  alternateHighlightText={['Incredible', 'Fantastic', 'Awesome']}
  alternateAnimation={true}
  alternateDuration={2000}
  highlightStyle="background"
  highlightColor="theme.success"
>
  This is Amazing
</Title>
```

**Animation Loop Control:**
```tsx
<Title
  animate={{
    from: { opacity: 0, transform: 'translateY(-20px)' },
    to: { opacity: 1, transform: 'translateY(0)' },
    duration: '1s',
    timingFunction: 'ease-out',
  }}
  animationLoop="infinite"
  highlightText="Infinite"
  highlightStyle="glow"
>
  Animation Loops Infinitely
</Title>
```

**Highlight Styles:**
- `background`: Solid background color
- `gradient`: Gradient background (requires `highlightSecondaryColor`)
- `underline`: Underline decoration
- `outline`: Text outline
- `glow`: Glowing text effect

### Form Components with Formik

**Basic Form:**
```tsx
import { Formik } from 'formik';
import * as Yup from 'yup';
import { FormikForm, FormikTextField, FormikCheckbox } from '@app-studio/web';

const schema = Yup.object({
  firstName: Yup.string().required('First name is required'),
  lastName: Yup.string().required('Last name is required'),
  email: Yup.string().email('Invalid email').required('Email is required'),
  agreeToTerms: Yup.boolean().oneOf([true], 'You must agree to the terms'),
});

<Formik
  initialValues={{ firstName: '', lastName: '', email: '', agreeToTerms: false }}
  validationSchema={schema}
  onSubmit={(values) => console.log(values)}
>
  {({ handleSubmit, isSubmitting }) => (
    <FormikForm>
      <Vertical gap={16} width="100%" maxWidth={400}>
        <FormikTextField name="firstName" label="First Name" placeholder="Enter your first name" />
        <FormikTextField name="lastName" label="Last Name" placeholder="Enter your last name" />
        <FormikTextField name="email" label="Email" placeholder="Enter your email" type="email" />
        <FormikCheckbox name="agreeToTerms" label="I agree to the terms and conditions" />
        <Button type="submit" onClick={handleSubmit} isLoading={isSubmitting}>
          Submit
        </Button>
      </Vertical>
    </FormikForm>
  )}
</Formik>
```

- `FormikTagInput` - Tag input
- `FormikChatInput` - Chat input with attachments

**Validation Patterns:**
```tsx
// Email validation
email: Yup.string().email('Invalid email format').required('Email is required')

// Password validation
password: Yup.string()
  .min(8, 'Password must be at least 8 characters')
  .matches(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/, 'Password must contain uppercase, lowercase, and number')
  .required('Password is required')

// Phone number validation
phone: Yup.string()
  .matches(/^\+?[1-9]\d{1,14}$/, 'Invalid phone number')
  .required('Phone number is required')

// URL validation
website: Yup.string().url('Invalid URL format').required('Website URL is required')

// Confirm password
confirmPassword: Yup.string()
  .oneOf([Yup.ref('password')], 'Passwords must match')
  .required('Please confirm your password')
```

## Best Practices for AI Agents

### 1. Always Use TypeScript Types
```tsx
import { ButtonProps, TitleProps, CardProps } from '@app-studio/web';

const MyButton: React.FC<ButtonProps> = (props) => {
  return <Button {...props} />;
};
```

### 2. Compose Components Logically
Group related components together and use proper hierarchy:
```tsx
// Good
<Card>
  <Card.Header>
    <Title>Header</Title>
  </Card.Header>
  <Card.Content>
    <Vertical gap={16}>
      <Text>Content</Text>
      <Button>Action</Button>
    </Vertical>
  </Card.Content>
</Card>

// Avoid
<Card>
  <Title>Header</Title>
  <Text>Content</Text>
  <Button>Action</Button>
</Card>
```

### 3. Use Semantic Prop Values
```tsx
// Good - using theme colors
<Button backgroundColor="theme.primary" color="color.white">Submit</Button>

// Avoid - using hardcoded colors
<Button backgroundColor="#0066cc" color="#ffffff">Submit</Button>
```

### 4. Handle Loading and Error States
```tsx
<Formik onSubmit={handleSubmit}>
  {({ handleSubmit, isSubmitting, errors }) => (
    <FormikForm>
      <FormikTextField name="email" label="Email" error={errors.email} />
      <Button type="submit" onClick={handleSubmit} isLoading={isSubmitting} disabled={isSubmitting}>
        {isSubmitting ? 'Submitting...' : 'Submit'}
      </Button>
    </FormikForm>
  )}
</Formik>
```

### 5. Maintain Consistent Spacing
Use the gap prop for consistent spacing:
```tsx
<Vertical gap={16}>  {/* 16px spacing */}
  <Title>Title</Title>
  <Text>Description</Text>
  <Button>Action</Button>
</Vertical>
```

### 6. Responsive Design
Use responsive props and media queries:
```tsx
<Title 
  size="xl" 
  responsive={true}  // Automatically adjusts based on screen size
  media={{
    mobile: { fontSize: 32 },
    tablet: { fontSize: 48 },
    desktop: { fontSize: 64 },
  }}
>
  Responsive Title
</Title>
```

### 7. Accessibility
Always provide labels and ARIA attributes:
```tsx
<FormikTextField 
  name="email" 
  label="Email Address"  // Always provide labels
  placeholder="Enter your email"
  aria-label="Email Address"
  required
/>
```

## Troubleshooting

### Common Issues

**Issue: Highlight animation not working**
- Ensure `highlightText` exactly matches text in children
- For arrays, each element must match individually
- Check that `highlightStyle` is set

**Issue: Form validation not triggering**
- Verify Yup schema is properly defined
- Check that field `name` prop matches schema keys
- Ensure `validationSchema` is passed to Formik

**Issue: Theme colors not applying**
- Use correct theme token format: `theme.primary`, `color.blue.500`
- Check ThemeProvider is wrapping your app
- Verify color exists in theme palette

**Issue: Animation not looping**
- Set `animationLoop="infinite"` for infinite loops
- Set `animationLoop={3}` for specific number of loops
- Ensure `animate` prop is properly configured
