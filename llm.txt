# App Studio Web Component Library

A comprehensive, accessible, and customizable React component library built with TypeScript and App Studio. This library provides 60+ production-ready UI components designed for building modern web applications with full type safety, accessibility support, and seamless form integration.

## Overview

App Studio Web is a complete React component library featuring:
- **TypeScript-first**: Built with TypeScript 4.9+ for complete type safety and IntelliSense support
- **40+ Core UI Components**: Comprehensive set of layout, form, navigation, feedback, and data display components
- **ADK Integration**: Specialized components for Agent Development Kit (ADK) integration
- **Formik Integration**: Seamless form handling with dedicated Formik wrapper components
- **Accessible**: WCAG-compliant components with proper ARIA attributes and keyboard navigation
- **Themeable**: Flexible theming system with light/dark mode support and customizable design tokens
- **Production Ready**: Battle-tested components used in real-world applications
- **Open Source**: MIT licensed and ready for AI-assisted development

## Installation

### NPM
```bash
npm install @app-studio/web
# or
yarn add @app-studio/web
```

### Manual Installation
```bash
# Clone the repository
git clone https://github.com/your-org/app-studio-web.git
cd app-studio-web

# Install dependencies
npm install

# Build the library
npm run build
```

## Theming and Customization

### Dark Mode
App Studio Web includes built-in support for light and dark themes:

```tsx
import { ThemeProvider } from 'app-studio';

function App() {
  return (
    <ThemeProvider theme={{ mode: 'dark' }}>
      {/* Your app */}
    </ThemeProvider>
  );
}
```

Individual components can override the theme mode:
```tsx
<Button themeMode="dark">Dark Button</Button>
```

### Theme System

The library uses a comprehensive theme system with:

#### Color Palette
- **Semantic Colors**: `theme.primary`, `theme.secondary`, `theme.success`, `theme.warning`, `theme.error`, `theme.info`, `theme.disabled`
- **Neutral Colors**: `color.white`, `color.black`, `color.gray.50` through `color.gray.900`
- **Color Scales**: `color.blue`, `color.green`, `color.red`, `color.yellow`, `color.purple` (50-900)

#### Typography
- **Font Family**: Inter/Geist
- **Font Sizes**: 12px to 48px
- **Font Weights**: 400 (regular) to 700 (bold)
- **Line Heights**: 1.2 to 1.8

#### Spacing
Uses a 4px grid system:
- 4px (extra small)
- 8px (small)
- 16px (medium)
- 24px (large)
- 32px (extra large)

#### Shapes
- `sharp`: No border radius (0px)
- `rounded`: Small border radius (4px)
- `pillShaped`: Pill shape (9999px)

#### Shadows
- `shadow.sm`: Small shadow
- `shadow.md`: Medium shadow
- `shadow.lg`: Large shadow
- `shadow.xl`: Extra large shadow

### Custom Styling

All components support style customization through the `views` prop:

```tsx
<Button
  views={{
    container: {
      backgroundColor: '#6200ee',
      borderRadius: 8,
    },
  }}
>
  Custom Button
</Button>
```

## Forms and Validation

### Formik Integration

The library provides dedicated Formik wrapper components for seamless form handling:

```tsx
import { Formik, Form } from 'formik';
import { FormikTextField, FormikCheckbox } from '@app-studio/web';
import * as Yup from 'yup';

const schema = Yup.object({
  email: Yup.string().email().required(),
  terms: Yup.boolean().oneOf([true]).required(),
});

function MyForm() {
  return (
    <Formik
      initialValues={{ email: '', terms: false }}
      validationSchema={schema}
      onSubmit={(values) => console.log(values)}
    >
      <Form>
        <FormikTextField name="email" label="Email" type="email" />
        <FormikCheckbox name="terms" label="I accept the terms" />
      </Form>
    </Formik>
  );
}
```

## State Management

Components use a combination of local and global state management:

### Local State
- Component-specific state managed via custom `useComponentNameState` hooks
- Encapsulates state logic and event handlers

### Global State (Zustand)
- `useModalStore`: Global modal state
- `useMessageStore`: Global message queue
- Used for state that needs to be shared across distant components

## API Integration

The library includes comprehensive API integration support with OpenAPI-generated services:

### Generated API Hooks
```tsx
import { UserService } from 'src/services/api';

function UserProfile({ userId }) {
  const getUserRequest = UserService.useUserControllerReadService({
    onSuccess: (data) => {
      console.log('User loaded:', data);
    },
    onError: (error) => {
      console.error('Error:', error);
    },
  });

  useEffect(() => {
    getUserRequest.run(userId);
  }, [userId]);

  return (
    <div>
      {getUserRequest.loading && <Loader />}
      {getUserRequest.data && <div>{getUserRequest.data.name}</div>}
    </div>
  );
}
```

See [API Integration Guide](./docs/api-integration.md) for complete documentation.

## Component Development

### File Structure
Each component follows a consistent structure:

```
ComponentName/
├── ComponentName.tsx                 # Main component file
├── ComponentName/                    # Inner folder
│   ├── ComponentName.props.ts        # Props interface
│   ├── ComponentName.state.ts        # State hook
│   ├── ComponentName.view.tsx        # Presentational component
│   ├── ComponentName.style.ts        # Style constants
│   └── ComponentName.type.ts         # Type definitions
└── examples/                         # Usage examples
    ├── default.tsx
    ├── variant.tsx
    └── index.ts
```

### Creating New Components

Use the provided script to scaffold new components:

```bash
npm run create-structure -- ComponentName
```
